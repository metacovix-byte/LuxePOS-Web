## 📍 Carrousel 8 — Backup OneDrive chiffré : pourquoi ça compte

> Vulgarisation tech sans condescendance.

### Slides

| # | Visuel | Texte slide |
|---|---|---|
| 1 | **Hook** : icône cadenas + cloud + carnet, fond crème. | "Pourquoi tes backups doivent être chiffrés." |
| 2 | Schéma simple : PC → OneDrive (chiffré) → toi seule peut lire. | "Tes données restent sur ton PC. ET dans ton OneDrive perso. Chiffrées AES-GCM 256." |
| 3 | Schéma anti-modèle : PC → serveur tiers → "qui sait ce qu'ils en font". | "PAS dans un serveur de startup qu'on ne connaît pas." |
| 4 | Texte explicatif : "Si OneDrive est piraté, mes données sont illisibles." | "Si OneDrive est piraté demain : tes ventes, clientes, coûts → illisibles sans ta clé locale." |
| 5 | Capture écran de la clé locale dans AppData. | "Ta clé est sur ton PC. Toi seule. Pas négociable." |
| 6 | **CTA** | "Sécurité par défaut → luxepos.app" |

### Caption

```
Petit cours technique express. Sans condescendance, promis.

Quand tu utilises un outil "dans le cloud" classique :
→ Tes données sont stockées sur leurs serveurs
→ Eux peuvent les lire (en théorie)
→ Si ils se font pirater = tes données fuitent en clair

Avec LuxePOS :
→ Tes données vivent sur TON PC (fichier .json dans AppData)
→ Un backup automatique part vers TON OneDrive perso
→ Mais ce backup est chiffré AVANT de quitter ton PC
→ Avec une clé qui reste SUR ton PC (jamais transmise)

Algo : AES-GCM 256. Le standard militaire actuel.

Conséquence concrète : si Microsoft se fait pirater demain, le
hacker récupère un fichier `LXP1\n[...]` illisible. Sans la
clé qui est chez toi, c'est du bruit binaire.

C'est ce que devrait faire toute app sérieuse. C'est rarement
le cas. Voilà.

Lien bio 💚
```

## Conseils généraux pour la publication

### Visuels

- **Format** : 1080×1350 px (4:5 vertical, occupe le max de feed)
- **Palette** : émeraude `#10b981`, crème `#fafaf7`, noir `#0f172a`, doré `#d4a574` en accent
- **Typo** : Playfair Display pour titres + Inter pour body
- **Logo** : losange émeraude en bas à droite de chaque slide, taille 40px
- **Slide 1 = hook fort** : 50 caractères max, lisible en 2 secondes
- **Slide finale = CTA simple** : un seul lien, un seul call

### Captions

- **Première ligne** : un hook fort (l'algo coupe à ~125 chars sans "Plus")
- **Ligne 3** : un blanc pour aérer
- **Bullet points** : maximum 5, espacés
- **CTA final** : toujours "Lien bio" (sauf si Reels où on peut taguer)
- **Signature** : 💚 en fin (pas d'autre emoji décoratif)

### Hashtags

- **Mix gros + niche** : 5 hashtags >100k posts + 5 entre 1k-100k + 5 ultra-niche <1k
- **Pas plus de 15 hashtags** par post (Insta privilégie maintenant 3-5 pertinents)
- **Premier commentaire** plutôt qu'en fin de caption (plus propre visuellement)
